# Security Policy

LidoPro is proprietary commercial software in active development.

Do not open public issues for sensitive vulnerabilities, private data exposure, credential leaks, payment concerns, or security-sensitive implementation details.

## Scope

Current public security scope covers repository/application code and local-first behavior.

The following systems are out of scope until explicitly implemented and released:

- hosted cloud infrastructure;
- real payment processing;
- public customer portal;
- production authentication/account systems;
- production SaaS infrastructure.

## Report Content

Do not submit real customer data, payment data, credentials, secrets, establishment data, private booking data, personal data, or private third-party material in any report.

Reports should include:

- a concise description;
- affected local application area;
- reproduction steps using local/dev data only;
- impact assessment;
- suggested mitigation if available.

## Contact

Until a dedicated security contact is published, contact the repository owner privately through a non-public channel. Do not disclose sensitive findings publicly before the owner has reviewed them.
