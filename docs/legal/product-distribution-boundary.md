# Product Distribution Boundary

This document defines distribution states for LidoPro.

## Development Source Repository

- Access: owner and explicitly permitted collaborators; public/source-available visibility may be enabled.
- Rights: review and development reference only unless separately authorized.
- Payments/cloud/accounts: not live unless explicitly implemented.
- Customer data: no real customer data in repository.
- Support terms: none.

## Internal/Local Development Build

- Access: owner development environment.
- Rights: internal development and testing.
- Payments/cloud/accounts: dev-only placeholders or local-only workflows unless explicitly released.
- Customer data: avoid real customer data; use fictional fixtures.
- Support terms: none.

## Private Evaluation Build

- Access: explicitly approved evaluators only.
- Rights: defined by written approval.
- Payments/cloud/accounts: disabled unless explicitly included in the evaluation scope.
- Customer data: only under approved privacy/security terms.
- Support terms: required if customer-facing.

## Commercial Customer Build

- Access: licensed customer or deployment target.
- Rights: defined by commercial license/customer terms.
- Payments/cloud/accounts: allowed only if implemented, provider-configured, reviewed, and released.
- Customer data: allowed only under customer terms, privacy policy, and data handling rules.
- Support terms: required.

## Public Commercial Release

- Access: public customers or marketplace distribution.
- Rights: defined by commercial license and product terms.
- Payments/cloud/accounts: allowed only if released and supported.
- Customer data: requires privacy policy, security posture, and operational support.
- Support terms: required.

## Hosted/SaaS Release

- Access: hosted service customers.
- Rights: defined by SaaS terms.
- Payments/cloud/accounts: allowed only with production infrastructure, provider agreements, security review, privacy policy, and support model.
- Customer data: requires production-grade data handling, backup, security, and incident processes.
- Support terms: required.
