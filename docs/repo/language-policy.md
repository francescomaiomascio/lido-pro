# Language Policy

This policy defines the canonical language rules for LidoPro repository-facing work and user-facing product copy.

## Repository-Facing Language

Use English for:

- `README.md`;
- technical documentation;
- commercial, repository, and legal documentation;
- code comments;
- repository-facing metadata and explanatory text.

Commit messages should preferably be written in English.

## Application UI Language

User-facing application UI is Italian-first. New UI copy should go through the existing i18n or product-copy structure where appropriate and should leave room for English and other languages.

Do not hardcode mixed Italian/English UI strings unless the mixed wording is an intentional canonical product term.

## Canonical Product And Domain Labels

Do not casually translate canonical product or module names. The following labels may remain Italian or hybrid in documentation and UI when used as product/domain terms:

- LidoPro
- Lido Studio
- Studio Mappa
- Lido Booking
- Lido Clienti
- Lido Listino
- Lido Pay
- Lido Cloud
- Layout attivo
- Registro

## Mixed-Language Rules

Avoid mixed Italian/English sentences in the same documentation paragraph unless the paragraph is explaining terminology.

`README.md` may include one concise Italian tagline or note near the top, but the main body must remain English.

## Historical Records

Do not rewrite old historical wave records only for language cleanup. Update historical records only when they create current public-facing confusion or incorrect product claims.
