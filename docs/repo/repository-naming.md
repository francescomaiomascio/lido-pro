# Repository Naming

The canonical public repository target name is:

```text
lido-pro
```

The product name is:

```text
LidoPro
```

## Historical Local Names

Older local development paths or historical documentation may still mention `beach-bdf`. Treat that as historical/local context only.

Product-facing documentation, commercial positioning, and current repository references should use LidoPro and `lido-pro`.

## Package And Platform Identifiers

- npm package name: `lido-pro`
- Tauri desktop identifier: `com.francescomaiomascio.lidopro`
- Capacitor Android app id: `it.lidopro.app`

Do not migrate Android package identifiers, app ids, or filesystem paths without a separate migration wave.

## Repository Boundary

The repository may be public/source-available, but it is not open source. Public visibility does not grant commercial use, redistribution, hosting, SaaS, customer deployment, or LidoPro brand/logo rights.
